# From-Scratch ANN Carpet Anomaly Detection

This project trains and evaluates a convolutional autoencoder for carpet defect
detection on the MVTec AD carpet dataset. The neural network is implemented from
scratch with NumPy: convolution, ReLU, sigmoid, average pooling, upsampling, MSE
loss, backpropagation, and Adam optimization are all in `src/nn_scratch.py`.

## Goal

The model learns to reconstruct only normal carpet images from
`data/carpet/train/good`. At test time, images with unusual reconstruction
behavior are treated as anomalies. This is an unsupervised anomaly detection
setup because defective training images are not used.

## Project Structure

```text
data/carpet/              MVTec carpet images and masks
src/nn_scratch.py         NumPy layers, loss, Sequential container, Adam
src/model.py              Convolutional autoencoder architecture
src/dataset.py            Image loading and mini-batch utilities
src/train.py              Training script and checkpoint/history saving
src/evaluate.py           Image metrics, patch scoring, pixel IoU, visualizations
outputs/checkpoints/      Saved model parameters
outputs/figures/carpet/   Evaluation plots
outputs/metrics/          JSON/CSV metrics generated by scripts
```

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Train

```bash
python3 src/train.py --epochs 20 --batch_size 4 --lr 0.001
```

This saves:

- `outputs/checkpoints/carpet_scratch_cae.pkl`
- `outputs/metrics/carpet_training_history.csv`
- `outputs/metrics/carpet_training_history.json`
- `outputs/figures/carpet/00_training_loss.png`

## Evaluate

```bash
python3 src/evaluate.py
```

This saves:

- `outputs/metrics/carpet_evaluation_metrics.json`
- `outputs/metrics/carpet_evaluation_records.csv`
- `outputs/metrics/carpet_threshold_sweep.csv`
- `outputs/metrics/carpet_pixel_threshold_sweep.csv`
- plots in `outputs/figures/carpet/`

The default anomaly score is now `max_patch`, inspired by the lecture case
study where large fabric images are divided into local patches. Instead of
averaging the reconstruction error over the full image, the evaluator finds the
highest-error local patch. This is useful because many defects are small and can
be diluted by global image MSE.

The default image threshold percentile is `90`. The earlier `95` percentile was
too conservative for this checkpoint: it reduced false alarms but missed many
defects. The default pixel threshold percentile is `99`, because lower pixel
thresholds over-mark large normal regions. The evaluator now saves threshold
sweep CSV files and plots so the threshold tradeoff is visible instead of hidden.

You can still reproduce the original global-MSE baseline:

```bash
python3 src/evaluate.py --score_type global_mse
```

Other score options are:

```bash
python3 src/evaluate.py --score_type topk_pixels --topk_percent 5
python3 src/evaluate.py --score_type max_patch --patch_size 8 --patch_stride 4
python3 src/evaluate.py --score_type topk_patches --patch_size 8 --patch_stride 4
```

The evaluator also uses the MVTec ground-truth masks to report pixel-level
localization metrics:

```text
pixel precision, pixel recall, pixel F1, pixel IoU
```

A diagnostic score-direction mode is also available:

```bash
python3 src/evaluate.py --score_mode auto
```

`auto` checks whether normal MSE or inverted MSE ranks labeled test defects
better. It is useful for analysis, but the final report should clearly state
that it uses test labels and therefore is not a pure unsupervised thresholding
method.

## Current Baseline Result

Using the existing checkpoint with global reconstruction MSE and the old 95th
percentile training threshold:

```text
Accuracy:  0.2991
Precision: 0.7059
Recall:    0.1348
F1-score:  0.2264
ROC-AUC:   0.2665
TP=12, TN=23, FP=5, FN=77
```

The low ROC-AUC indicates that this checkpoint often reconstructs defective
images with lower MSE than normal test images. That is an important discussion
point: the implementation is from scratch, but the current model and scoring
strategy need further tuning to become a strong detector.

Using the lecture-inspired local patch score on the same checkpoint:

```text
Score type: max_patch
Threshold percentile: 90
Accuracy:  0.3590
Precision: 0.6591
Recall:    0.3258
F1-score:  0.4361
ROC-AUC:   0.3772
Pixel IoU: 0.0667
```

This is still not a strong detector, but it improves the fixed-threshold
image-level F1-score by focusing on local high-error regions instead of
averaging defects away. The low ROC-AUC shows that score ranking is still weak,
so the next modeling step should be denoising training or better threshold
calibration rather than claiming the detector is solved.

## Suggested Improvements

- Train for more epochs and compare multiple learning rates.
- Try different image sizes such as 32, 64, and 96 if runtime allows.
- Add a validation split from normal training images for threshold selection.
- Compare vanilla CAE with denoising CAE.
- Improve the localization mask with smoothing or connected-component filtering.
- Add optional clustering or classification on the encoder feature vectors.
